Run setup

The CLI detects the existing app and writes public configuration.

Add the public key

Sparkle keeps the private key in Keychain; only the public EdDSA key enters the app.

Review and apply

Inspect every path, then generate the integration with timestamped backups.

Finish two Xcode steps

Add the official package and keep the updater alive from app startup.

Verify and stage the release

Doctor, Release build, architecture and code-signing inspection, exact EdDSA verification, checksums, manifest generation, and an older-to-newer test update complete the evidence.

Built for coding agents

A protocol, not a vague prompt.

`AGENTS.md`, `llms.txt`, JSON Schema, exact commands, safety boundaries, expected files, and a completion report let an AI integrate the toolkit without guessing your release security.

Read the AI protocol
Security boundary

Signing secrets never belong in project configuration.

The toolkit separates public metadata from private signing material, rejects suspicious configuration, minimizes workflow permissions, and verifies the artifact users will actually install.
