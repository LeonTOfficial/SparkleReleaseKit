A short path from Xcode app to secure update.

Follow the guided setup, review every generated change, and verify the actual release archive.

Requirements

Use macOS 13 or later, a current stable Xcode installation, Git, an existing macOS app target, and a shared scheme. A paid Apple Developer membership is optional; Sparkle EdDSA authentication is required in every release mode.

Install the toolkit

Download the latest tested macOS package, extract it, open Terminal in its SparkleReleaseKit folder, then run:

./install.sh
sparklekit version

Every release includes a SHA-256 checksum and GitHub artifact attestation. If macOS quarantines the currently non-notarized CLI, build it locally instead:

git clone https://github.com/LeonTOfficial/SparkleReleaseKit.git
cd SparkleReleaseKit
./scripts/bootstrap.sh
./scripts/install.sh
Installation is user-local by default. SparkleReleaseKit does not need administrator access.

Choose a release mode

freeSparkle EdDSA stays mandatory. Ad-hoc app signing is accepted; Developer ID and notarization are reported as optional.
developer-idRequires Developer ID, Hardened Runtime, Gatekeeper acceptance, and a valid notarization staple.
autoInspects the artifact and reports the strongest complete route without acquiring credentials or submitting to Apple.
Developer ID is not a Sparkle update signature. It is a separate, optional Apple trust layer.
Free distributionDeveloper ID distribution

Detect and configure

sparklekit setup "/path/to/YourApp" --release-mode free

Inspect the generated sparklekit.json. Generate the Sparkle EdDSA key once and add only its public key to updates.publicEDKey.

sparklekit integrate "/path/to/YourApp"
sparklekit integrate "/path/to/YourApp" --apply

The first command is always a preview. Apply mode creates a timestamped backup before replacing a managed file.

Finish in Xcode

  1. Add https://github.com/sparkle-project/Sparkle with Swift Package Manager.
  2. Add the Sparkle product and generated updater file to the app target.
  3. Keep AppUpdater.shared alive from startup.
  4. Add a Check for Updates command.

The generated SparkleReleaseKit/INTEGRATION.md contains code matched to the detected app style.

Verify before publishing

sparklekit doctor "/path/to/YourApp" --json
sparklekit test "/path/to/YourApp" --json
sparklekit verify "/path/to/YourApp.zip" --project "/path/to/YourApp" --release-mode free --json
sparklekit verify-update "/path/to/YourApp.zip" --appcast "/path/to/appcast.xml" --version 120

Structured reports use stable severity, detail, and remediation fields. The final command cryptographically verifies that the appcast signature authenticates the exact archive bytes.

Prepare the signed feed

sparklekit prepare-release "/path/to/YourApp-1.2.0.zip" \
  --project "/path/to/YourApp" \
  --version 1.2.0 \
  --notes "/path/to/release-notes.md" \
  --release-mode free \
  --generate-appcast "/path/to/Sparkle/bin/generate_appcast"

The official Sparkle tool reads the private key from macOS Keychain. Review the isolated stage, validate its feed, and test an older real app build before publication.

Use a coding agent

Ask the agent to read AGENTS.md and AI/AI_INTEGRATION.md first. The protocol defines the safe edit boundary, exact commands, prohibited secret handling, and required completion evidence.
